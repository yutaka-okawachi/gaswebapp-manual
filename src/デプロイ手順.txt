### A) 自動デプロイスクリプトを使う方法（推奨）

PowerShellでプロジェクトフォルダに移動し、`-env` で環境を指定して実行します。

# 1. テスト環境へのデプロイ
.\deploy.ps1 -env staging -d "テスト用の修正内容"

# 2. 本番環境へのデプロイ
.\deploy.ps1 -env production -d "本番リリースの内容"


### B) 手動でデプロイする方法

手動でのデプロイは、バージョン番号の指定ミスやIDのコピーミスが起こりやすいため、自動スクリプトの利用を推奨します。

# 1. ローカルの変更を反映
clasp push

# 2. 新しいバージョンを作成し、番号を控える (例: 567)
clasp version "検索ボタンのUIを修正"

# 3. 控えたバージョン番号で、指定したい環境のIDを使ってデプロイ
# 本番用ID: AKfycbxWl-KLwo8SnOyqQT84gJyrofRQnIp_GBv8Pg0N5athPAoxp9LBuwj0HDTXkFqh0xiGsw
# テスト用ID: AKfycbwgYpxpOQ3kez3T_5AAB6b4eFGy2yhjqcc0r84o4NjVVfq8B54RXLRikVo0rgwhtW-eyQ
clasp deploy -i [ここにデプロイIDを貼り付け] -V 567 -d "検索ボタンのUIを修正"
