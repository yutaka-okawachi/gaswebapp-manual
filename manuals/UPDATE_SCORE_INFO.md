# 楽譜情報の更新手順マニュアル

このドキュメントでは、ワーグナーおよびシュトラウスの作品検索画面に表示される「楽譜情報（出版社、プレート番号、IMSLPリンク等）」を更新する手順を説明します。

## 概要

楽譜情報は、以下の2箇所にある同一のメタデータファイルによって管理されています。
1. `src/score_metadata.js` (サーバー側: GAS用)
2. `mahler-search-app/js/score_metadata.js` (クライアント側: GitHub Pages用)

情報を変更する際は、この**両方のファイルを同期させる**必要があります。

Google スプレッドシートの通常データだけを修正する場合、この手順は不要です。
この手順は、出版社、プレート番号、IMSLP リンクなど、検索画面に表示する楽譜情報をコード側で修正する場合に使います。

## 更新手順

### 1. メタデータファイルの編集
`src/score_metadata.js` を開き、該当する作品の情報を編集します。

```javascript
    "feen": {
        "title": "妖精 (Die Feen)",
        "workNo": "WWV 32",
        "publisher": "出版社名",
        "plate": "プレート番号",
        "edition": "版情報・補足",
        "imslp": "IMSLPのURL"
    },
```

- **ID (`feen` 等)**: プログラム内部で使用されるIDです。変更しないでください。
- **各項目**: HTMLエスケープ処理が行われるため、通常の文字列として記述して問題ありません。

### 2. クライアント側ファイルへのコピー
編集したファイルを `mahler-search-app/js/score_metadata.js` に上書きコピーします。

**Windows (PowerShell/Cmd) の場合:**
```powershell
copy src\score_metadata.js mahler-search-app\js\score_metadata.js
```

コピー後、2つのファイルが同じ内容になっていることを確認してください。

### 3. 同期とデプロイ
変更を保存したら、プロジェクトルートにある同期スクリプトを実行して、GASおよびGitHub Pagesへ反映させます。

```powershell
.\sync-data.ps1
```

このスクリプトにより以下の処理が自動で行われます：
1. GASサーバーへのコード送信 (`clasp push`)
2. ローカル変更のGitコミット
3. GitHubへのプッシュと公開

## 注意事項

- **正規化されたID**: メタデータのキー（`walkuere` など）は、検索ロジックで正規化された文字列と一致している必要があります。新規作品を追加する場合は注意してください。
- **IMSLPリンク**: リンクは `<a>` タグとして出力され、別タブで開く設定 (`target="_blank"`) になっています。
- **ロールバック**: もし表示が崩れたりエラーが出た場合は、Gitで以前のバージョンに戻して `.\sync-data.ps1` を再実行してください。
- **後継者向け**: PC操作に詳しくない場合は、このファイルを直接編集せず、前任者または技術に詳しい協力者に依頼してください。
