# 長期保守・継承マニュアル (Legacy Maintenance & Succession Guide)

## 1. はじめに
本ドキュメントは、本プロジェクト（音楽用語検索アプリ）の製作者が不在となった場合、あるいは信頼できる後継者にプロジェクトを完全に引き継ぐ場合に必要となる情報をまとめたものです。

本システムは Google スプレッドシートをデータベースとし、GitHub Pages で公開されています。データの継承を適切に行うことで、サイトを長期的に維持することが可能です。

PC 操作に詳しくない後継者には、まず [継承者の方へ - 最初に読むページ](./SUCCESSOR_START_HERE.md) を案内してください。
前任者が事前に整理すべき情報は [継承情報チェックリスト](./SUCCESSOR_INFO_CHECKLIST.md) にまとめています。

---

## 2. プロジェクト資産のリスト
後継者が管理すべき主要な資産は以下の通りです。

| 資産 | 役割 | 場所/URL |
|---|---|---|
| **Google スプレッドシート** | マスターデータ（用語、メタデータ） | [Google Drive 内のスプレッドシート] |
| **Google Apps Script (GAS)** | データ処理、HTML生成、GitHub同期API | スプレッドシートの「拡張機能 > Apps Script」 |
| **GitHub リポジトリ** | ソースコード、公開サイトのホスティング | `https://github.com/yutaka-okawachi/gaswebapp-manual` |
| **ドメイン / Web サイト** | 公開されているアプリケーション | `https://yutaka-okawachi.github.io/gaswebapp-manual/` |

## 3. アカウントとデータの継承手順

### 3-1. Google アカウントの継承 (推奨設定)
Google が提供している **「アカウント無効化管理ツール」** を設定しておくことで、一定期間アクセスがない場合に、信頼できる人物へデータのダウンロード権限を自動的に付与したり、アカウントの通知を送ったりすることができます。

1.  [Google アカウント無効化管理ツール](https://myaccount.google.com/inactive) にアクセスします。
2.  **待機期間**（例：3ヶ月または6ヶ月）を設定します。
3.  **通知する連絡先** に、後継者のメールアドレスを登録します。
4.  **共有するデータ** として「Google ドライブ」と「Google スプレッドシート」を必ず選択してください。

### 3-2. スプレッドシートの共同編集者の追加
存命中に後継者が決まっている場合は、スプレッドシートの「共有」ボタンから、後継者の Google アカウントに **「編集者」** 権限を付与してください。これにより、後継者の Google ドライブからもアクセス可能になります。

### 3-3. GitHub の継承
1.  **共同作業者の追加**: GitHub リポジトリの `Settings > Collaborators` から後継者を招待し、Write（書き込み）権限を付与します。
2.  **アカウントの継承設定 (Successor)**: GitHub の `Settings > Account > Successor settings` にて、後継者の GitHub ユーザー名を指定しておくと、製作者に万が一のことがあった際にリポジトリの管理権限を譲渡する手続きが可能になります。

## 4. アクセストークン等の機密情報の管理

本システムの自動同期には、2種類のトークンが使用されています。これらはセキュリティのため、リポジトリ（コード）内には直接記載されていません。

### 4-1. GitHub Personal Access Token (PAT)
GAS から GitHub にデータを書き込むために必要です。
*   **保管方法**: パスワードマネージャーや、信頼できる人へのメモとして安全に保管してください。
*   **再発行**: 期限が切れたり紛失した場合は、[GitHub の設定](./SETUP_GUIDE.md#6-github-連携の設定-初回およびトークン更新時) から再発行し、スプレッドシートの「🔧 GitHub同期」メニューから再登録する必要があります。

### 4-2. GAS 認証トークン (GAS_SECRET_TOKEN)
ローカル PC と GAS ウェブアプリの間で通信を認証するための合言葉です。
*   **確認場所**: GAS エディタの「プロジェクトの設定 > スクリプトプロパティ」にある `GAS_SECRET_TOKEN` の値です。
*   **後継者への伝達**: 後継者が自分の PC で `sync-data.ps1` を動かす際、`.env` ファイルにこの値を記述する必要があります。

## 5. 後継者による運用・更新フロー

後継者が作業を引き継ぐための最小限のステップです。

1.  **情報の受け渡し**: 前任者は [継承情報チェックリスト](./SUCCESSOR_INFO_CHECKLIST.md) に沿って、Google スプレッドシート、GitHub、GAS スクリプトプロパティの情報を整理します。
2.  **作業フォルダの準備**: PC 操作に詳しくない後継者には、前任者がプロジェクトのフォルダ一式を渡し、`ドキュメント\gaswebapp-manual` など分かりやすい場所に置いてもらいます。
3.  **対話式セットアップ**: 後継者は `01_START_SUCCESSOR_SETUP.bat` をダブルクリックします。PowerShell を自分で開く必要はありません。
4.  **認証**: `clasp login` で、前任者から指定された Google アカウントまたは共有済みの後継者アカウントでログインします。
5.  **同期の確認**: `02_RUN_SYNC.bat` をダブルクリックし、正しく動作することを前任者と一緒に確認します。

コマンド操作に慣れている後継者の場合は、[セットアップガイド](./SETUP_GUIDE.md) に従って `git clone` から始めても構いません。

---

## 6. トラブル時の連絡とサポート
本プロジェクトは、Google Apps Script と GitHub という無料枠の範囲内で運用されています。仕様変更等で動作しなくなった場合は、以下の情報を参考にしてください。

*   **GAS のログ確認**: GAS エディタの「実行数」から、エラーメッセージを確認できます。
*   **AI アシスタントの活用**: 本リポジトリの構成（`sync-data.ps1` の仕組みなど）を AI（ChatGPT, Claude, Gemini 等）に読み込ませることで、不具合の修正方法を提示してもらうことが可能です。
