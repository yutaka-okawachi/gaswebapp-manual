# 継承者の方へ - 最初に読むページ

このプロジェクトは、音楽用語検索サイトを維持するためのデータ、プログラム、手順書をまとめたものです。
主なデータは Google スプレッドシートにあり、公開サイトは GitHub Pages で表示されています。

PC操作に詳しくない場合は、このページだけを上から順に確認してください。

## 全体の流れ

後継作業では、次の3つを順番に確認します。

1. 前任者から必要な権限と情報を受け取る。
2. 自分の PC に、このプロジェクト用の作業フォルダを置く。
3. セットアップ用の画面を起動し、質問に答える。

分からない質問が出た場合は、無理に進めず、前任者または管理者に確認してください。

## まず行うこと

1. このプロジェクトの管理者から、必要なアカウント権限と情報を受け取ります。
2. 前任者から、このプロジェクトのフォルダ一式を受け取ります。ZIP ファイルで受け取った場合は、右クリックして「すべて展開」を選びます。
3. Windows PC の分かりやすい場所に作業フォルダを作ります。
4. 受け取ったプロジェクトのフォルダを、作業フォルダの中に置きます。
5. プロジェクトのフォルダを開き、`scripts` フォルダ内の `start-successor-setup.bat` をダブルクリックします。

おすすめの置き場所は次のどちらかです。

```text
ドキュメント\gaswebapp-manual
```

または

```text
C:\Users\自分のユーザー名\gaswebapp-manual
```

PowerShell で直接実行する場合は、このプロジェクトのフォルダで次を実行します。

```powershell
.\scripts\setup-successor.ps1
```

PowerShell でスクリプト実行が禁止されていると表示された場合は、同じ画面で次を実行してから、もう一度上のコマンドを実行してください。

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

## セットアップ画面での答え方

セットアップ画面では、質問の最後に次のような表示が出ます。

```text
[y=はい / N=いいえ]
```

この場合は、`y` を入力して Enter を押すと「はい」、`n` を入力して Enter を押すと「いいえ」です。
大文字の `N` が表示されている質問は、Enter だけを押すと「いいえ」になります。

インストールやログインの質問で迷った場合は、いったん `n` を選んで止めてください。

## 入力を求められる情報

セットアップ中に、以下を入力する場面があります。

| 入力するもの | どこで使うか | 誰から受け取るか |
|---|---|---|
| GAS Web App URL | `.env` の `GAS_DEPLOY_URL` | 前任者または管理者 |
| GAS_SECRET_TOKEN | `.env` の `GAS_SECRET_TOKEN` | 前任者または管理者 |
| Google アカウント | `clasp login` | 自分のアカウント、または引き継いだ管理アカウント |
| GitHub アカウント | リポジトリ更新 | 自分の GitHub アカウント |

`.env` は、この PC だけで使う秘密情報のメモファイルです。
GitHub には公開しません。
セットアップ画面が必要な値を聞くので、前任者から受け取った値を貼り付けてください。

GitHub Personal Access Token は、できれば前任者のものを受け取らず、自分の GitHub アカウントで新しく発行してください。
詳しい手順は [環境構築ガイド](SETUP_GUIDE.md) の「GitHub 連携の設定」を見てください。

## よく出る言葉

| 言葉 | このプロジェクトでの意味 |
|---|---|
| Google スプレッドシート | 用語データが入っている表。普段の修正はここで行います。 |
| GAS / Google Apps Script | スプレッドシートのデータを処理して、サイト用データを作る仕組みです。 |
| GitHub | サイトのファイルを置いて公開する場所です。 |
| clasp | PC から Google Apps Script にログイン・更新するための道具です。 |
| `.env` | この PC だけで使う秘密情報のメモファイルです。 |
| token / トークン | パスワードのように扱う秘密の文字列です。人に見せないでください。 |

## ブラウザが開いた場合

`clasp login` を実行すると、ブラウザで Google のログイン画面が開きます。
このときは、前任者から指定された Google アカウント、または自分の Google アカウントでログインし、表示される許可画面を進めてください。

どの Google アカウントを使えばよいか分からない場合は、許可せずに前任者へ確認してください。

## 作業フォルダについて

`clasp` や `sync-data.ps1` は、プロジェクトのフォルダの中で使います。
そのため、後継者には「このフォルダを作業場所として使う」と決めてもらう必要があります。

フォルダ名や置き場所は自由ですが、次のような場所は避けてください。

- USB メモリの中
- OneDrive の同期中にエラーが出やすい場所
- 日本語や空白が非常に多い長いパス
- 管理者権限が必要な `C:\Program Files` などの場所

迷った場合は、前任者がフォルダを丸ごと用意し、後継者にはそのフォルダを開いてもらうのが安全です。

GitHub から取得する方法もありますが、PC操作に慣れていない場合は、前任者からフォルダ一式を受け取る方法を推奨します。

## 日常的な更新方法

通常は Google スプレッドシートの内容を編集し、必要に応じて次を実行します。

```powershell
.\sync-data.ps1
```

成功すると、GitHub Pages の公開サイトに反映されます。

`.env` に入れる値やトークンは秘密情報です。
GitHub、メール、公開チャットなどには貼り付けないでください。

前任者から渡された情報が足りない場合は、[継承情報チェックリスト](SUCCESSOR_INFO_CHECKLIST.md) の「後継者に渡す情報」を確認してください。
